## Simulated Incidnet Scenario

A controlled incident scenario was created on the WindowsTarget virtual machine to generate forensic artifacts that could be acquired and analyzed later.

During the simulation, several user activities were performed including downloading a file, renaming it, moving it to the Downloads directory, opening the file, and deleting it. 
Additional system activity was generated by executing PowerShell commands such as whoami, ipconfig /all, and netstat -an. Several websites were also visited to create network traffic.

These actions were intended to simulate suspicious or investigative activity and produce disk and network artifacts that could be collected using forensic acquisition tools.

## Disk Image Acquisition

A partial forensic disk image of the Kali forensic workstation was created using the dd utility. Investigators use this to preserve digital evidence without alerting the original data.

Command used:

sudo dd if=/dev/sda of=Phase2/kali_disk_image.dd bs=1M count=1024

This command copies the first portion of the system disk and stores it as a forensic disk image file for analysis.

Hash verificaion command:

## Artifact Extraction and Analysis

After acquiring the partial forensic disk image, artifact extraction was performed using Bulk Extractor.

Command used: 

bulk_extractor Phase2/kali_disk_image.dd -o Phase2/artifacts

Bulk Extractor scans disk images and automatically extracts forensic artifacts such as email addresses, URLs, network activity, and file fragments that may indicate user behavior or system activity.

The tool generated multiple artifact files including:

- email.txt
- httplogs.txt
- url.txt
- winprefetch.txt
- telephone.txt
- facebook.txt
- sqlite_carved.txt

These artifacts provide investigators with potential indicators of user activity and system usage during the simulated incident.

Example artifact review command:
cat Phase2/artifacts/url.txt | head

The extracted artifact files confirm that forensic data can be recovered from the disk image, demonstrating the effectiveness of artifact extraction
during forensic investigations.


## Network Traffic Capture

Network traffic between the forensic workstation and the target system was captured using tcpdump.

Command used:

sudo tcpdump -i -eth0 -w Phase2/network_capture.pcap

This capture recorded network communication within the Azure virtual network enviornment and preserved it for later analysis.

## Chain of Custody
 
All evidence collected during the investigation was preserved and documented to maintain data integrity.

The following steps were taken to ensure proper evidence handling:

- Disk acquisition was performed using dd to create a forensic disk image.
- SHA256 hashing was used to verify the integrity of the disk image.
- Network traffic was captured using tcpdump
- Azure disk snapshot was created to preserve the WindowsTarget system.
- Extracted artifacts were analyzed using Bulk Extractor

Evidence files were retained locally on the Kali forensic workstation. Due to GitHub file size limitations, large evidence files such as disk images and packets were excluded from 
repository tracking and documented instead.


## Challenges and Mitigation

During the investigation several technical challenges were encountered. 

Kali Virtual Machine Deletion
The original Kali forensic workstation was accidentally deleted in Azure. A new Kali VM was deployed and the forensic environment was rebuilt.

Large Evidence Files and GitHub Limitations
GitHub rejected large evidence files such as disk images and packet captures due to the 100 MB file size limit. These files were retained locally on the Kali forensic workstation
and excluded from the repository using .gitignore.

Total Compatibility Issues
Some forensic tools such as Volatility could not be installed due to package conflicts within the Azure Kali environment. Alternative tools such as Bulk Extractor were used to contine artifact extraction.
